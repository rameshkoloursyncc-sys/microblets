ROOT CAUSE IDENTIFIED:
----------------------
The sync logic in SmartStockAlertService was correct, BUT it only ran:
1. When dashboard loaded
2. When "Send Smart Excel Report" was clicked
3. NOT when user did IN/OUT transactions from frontend

This caused STALE DATA in tracking table, leading to wrong calculations.

THE REAL PROBLEM:
-----------------
When user updates stock from frontend:
1. Controller updates product table (balance_stock, ribs, meter, total_mm)
2. Controller creates InventoryTransaction record
3. Controller tries to reset alert BUT only if alert_sent = true
4. Tracking table NOT updated with new stock values
5. Later when sync runs, it uses OLD tracking data as baseline
6. Wrong deficit calculation results

EXAMPLE (A18 Scenario):
- OUT 29 → balance_stock = 71, but tracking not updated
- Sync runs → Creates tracking with current_stock = 71
- Alert sent → last_alerted_stock = 71, previous_stock = 71
- OUT 60 → balance_stock = 11, but tracking still shows current_stock = 71
- Sync runs → Sees previous_stock = 71 (WRONG, should be from tracking)
- Calculates deficit from OLD data → WRONG RESULT

THE SOLUTION:
-------------
Update tracking table IMMEDIATELY in controllers when stock changes.
This ensures tracking is always in sync with actual stock.

WHAT WAS CHANGED:
-----------------
Replaced simple reset logic:
```php
if ($tracking && $tracking->alert_sent) {
    $tracking->resetAlert();
}
```

With REAL-TIME tracking update:
```php
if ($tracking) {
    // Stock above min → Reset
    if ($newStock >= reorder_level) {
        Update: current_stock, previous_stock, dies_needed=0, 
                alert_sent=false, last_alerted_stock=null
    }
    // Stock below min → Update tracking
    else {
        // IN transaction → Update stocks
        if ($newStock > $previousStock) {
            Update: current_stock, previous_stock
        }
        // OUT transaction → Calculate deficit
        else if ($newStock < $previousStock) {
            // Incremental alert needed
            if (alert_sent && newStock < last_alerted_stock) {
                deficit = last_alerted_stock - newStock
                Update: current_stock, previous_stock, dies_needed, alert_sent=false
            }
            // First time alert
            else if (!alert_sent) {
                deficit = reorder_level - newStock
                Update: current_stock, previous_stock, dies_needed, alert_sent=false
            }
            // Stock dropped but no new alert
            else {
                Update: current_stock, previous_stock
            }
        }
    }
}
```
